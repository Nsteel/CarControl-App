Setup

There are two ways to set up the app to be able to connect to a model car. You can use a simulated car, or a real model car.

Using a Simulated Model Car

It is likely easier to access a simulated car rather than a real one, so follow these steps to set up a simulation

Step 1: Installing Ubuntu 16.04

Download a 32 or 64 bit Ubuntu 16.04 iso, either will work, and install it on a physical computer or a virtual machine. The model car will move much much smoother if Ubuntu 16.04 is installed to a physical computer.

Note: If installing Ubuntu on a virtual machine, make sure that you use a Bridged Network Adapter and NOT a NAT adapter.

Step 2: Installing ROS Kinetic

Once you have Ubuntu installed, follow these instructions to install ROS Indigo on your machine.

http://wiki.ros.org/kinetic/Installation/Ubuntu

Step 3: Installing the PSES-UcBridge package

 1. Open a terminal and run the following command in the src folder of your catkin workspace git clone https://github.com/tud-pses/pses_ucbridge.git
 2. cd ..
 3. Compile it! catkin_make

Step 5: Launching the Simulator

To begin open a terminal and run the command ifconfig, then run the command export ROS_IP="the IP address of wlan0". Once you have done that, start the simulator in the same terminal using the command roslaunch pses_simulation simulation.launch.

Step 6: Connecting to the Simulated Car

To connect to the simulator, make sure that your computer running the simulator and your Android device are on the same wifi network first. Then, open the app and click the add a car button. When the prompt pops up, replace only the word "localhost" with the same IP address that you set your ROS_IP environment variable to earlier. Hit connect and you should connect to the simulated car and be able to use the app.

Using the real Model Car

Connecting to a model car will be easier because everything was pre-installed for you.

Step 1: Connecting to the Model Car

To begin open a terminal and run the command ifconfig on the car's mainboard, then run the command export ROS_IP="the IP address of wlan0". Once you have done that, start the launch file in the same terminal using the command roslaunch pses_ucbridge ucbridge.launch.

To connect to the model car, make sure your Android device is on the car's wifi network. Then open the app and add a car. Replace only the word "localhost" with the IP address of the car and hit connect. You should connect to the car and be able to use the app.
